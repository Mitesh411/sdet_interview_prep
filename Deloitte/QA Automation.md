- Difference between Smoke, Sanity, Regression, Exploratory testing.

| Type                    | Purpose                                                     | When It’s Done                        | Scope                |
| ----------------------- | ----------------------------------------------------------- | ------------------------------------- | -------------------- |
| **Smoke Testing**       | To check if the build is stable enough for further testing  | After a new build is deployed         | Shallow and broad    |
| **Sanity Testing**      | To verify specific bug fixes or small changes               | After minor updates or bug fixes      | Narrow and focused   |
| **Regression Testing**  | To ensure new changes didn’t break existing functionality   | Before release or after major updates | Deep and wide        |
| **Exploratory Testing** | To discover unexpected issues without predefined test cases | Anytime, especially early stages      | Creative, unscripted |



👉 In short: Smoke = build stability, Sanity = fix verification, Regression = full system safety, Exploratory = human curiosity testing.
